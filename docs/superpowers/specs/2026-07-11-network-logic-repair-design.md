# 校园网保活逻辑修复设计

## 目标

修复托盘刷新、网络状态误判、认证器错误锁定、恢复动作跨网卡、看门狗重启失效和双状态机漂移问题，同时保留现有命令行与托盘入口。

## 架构

`network_monitor` 负责产生可信的结构化快照；`KeepAlive` 作为唯一恢复策略控制器；托盘只展示控制器状态并转发人工登录、登出和退出操作。认证路由在网络在线时不猜测学校类型，只在捕获到认证重定向后选择并缓存认证器。

## 关键行为

- HTTP 302 不再表示外网在线，而是认证重定向候选。
- Windows 英文 `Disconnected` 不得匹配为 `Connected`。
- 防抖只累计相同异常状态，状态变化后重新计数。
- Wi-Fi 和以太网恢复动作必须绑定目标接口。
- Wi-Fi 兜底在线时，以太网连续稳定后才尝试切回；失败则恢复 Wi-Fi。
- 托盘格式化方法和退出标志必须始终存在。
- 看门狗重启前释放单实例锁，避免新进程被旧进程拒绝。
- 登录前不再解析特定学校域名。

## 错误处理

网络探测失败返回 `UNKNOWN`，不触发破坏性恢复；恢复动作返回明确结果，由控制器统一维护异常计数、退避和显示状态。看门狗按操作阶段刷新心跳，正常长恢复不会被误判为卡死。

## 测试

增加网络响应、英文网卡状态、认证延迟选择、托盘生命周期、相同状态防抖、接口绑定恢复、网线切回和单实例重启测试。每项修复遵循先失败、后实现的 TDD 流程，并在最后运行完整 pytest 与 compileall。
